# Patch Notes — v1.1.0 (2025-03-13)

## 덱 추천 스코어링 정규화 및 메트릭스 검증

### 요약

멀티 속성(듀얼 컬러/듀얼 플레이스타일) 덱이 구조적으로 과다 추천되던 문제를 수정하고, 실제 유저 데이터(Umami) 기반 검증 테스트를 추가했습니다.

---

## 문제 (Before)

스코어링 엔진에서 `colors[]`와 `playstyle[]` 배열을 `reduce(sum)`으로 합산하는 방식 때문에, 속성이 2개인 덱이 1개인 덱 대비 약 **2.5배** 색상 점수 우위를 가졌습니다.

| 덱                    | 티어 | 프로덕션 Top-1 비율 |
| --------------------- | :--: | :-----------------: |
| 자흑 루피 (듀얼)      |  B   |      **17.0%**      |
| 적청 마르코 (듀얼)    |  B   |      **10.7%**      |
| 자 루피 (싱글)        |  S   |        3.4%         |
| 적황 벨로 베티 (싱글) |  S   |        2.6%         |
| 녹 보니 (싱글)        |  A   |        0.4%         |

→ B티어 듀얼 덱이 S/A티어 싱글 덱보다 훨씬 많이 추천되는 비정상적 분포

---

## 수정 내용

### 1. 속성 정규화 (`scoring.ts`)

컬러/플레이스타일 점수를 속성 개수로 나눠 듀얼 속성의 구조적 이점을 제거.

```typescript
// Before
const colorScore = deck.colors.reduce((sum, c) => sum + (weights.colors[c] ?? 0), 0);

// After
const colorScore =
  deck.colors.reduce((sum, c) => sum + (weights.colors[c] ?? 0), 0) / deck.colors.length;
```

### 2. 티어 가중치 조정 (`scoring.ts`)

메타 정렬도를 높이기 위해 티어 멀티플라이어를 소폭 상향.

```
tier: 1.0 → 1.3
```

> 과도한 티어 편향을 방지하기 위해 보수적 수치 적용 (1.5 이상은 의도적으로 배제)

### 3. 테스트 픽스쳐 수정 (`recommend.test.ts`)

컨트롤 선호 테스트에서 모순되는 답변 조합(control + top-meta) 수정.

```
q4-meta: "q4-top" → "q4-counter"
```

S티어 컨트롤 덱이 존재하지 않아 테스트가 의미 없는 조합을 검증하고 있었음.

## 검증

### 전수 조합 메트릭스 (`metrics.test.ts` — 신규)

- **11,340개** 유효 답변 조합 전수 시뮬레이션
- 커버리지: 모든 덱이 최소 1회 이상 추천됨
- 지배도: 단일 덱 Top-1 점유율 60% 미만
- 멀티/싱글 비율: 0.95x (수정 전 1.21x → 정상 범위)

### 실 유저 가중 메트릭스 (`metrics.test.ts` — 신규)

- Umami 프로덕션 데이터 **771건** 추출
- **167개** 고유 답변 코드 + 발생 빈도 가중치 반영
- 가중 Top-3 분포: 단일 덱 60% 미만
- 가중 Top-1 분포: 단일 덱 35% 미만
